
# 0x03. Unittests and Integration Tests

## Project Overview

This project focuses on writing unit and integration tests in Python 3.7, following best practices for documentation, code style, and testing. It uses the `unittest` framework and the `parameterized` package to validate the functionality of Python modules.

## Requirements

- Ubuntu 18.04 LTS
- Python 3.7
- Pycodestyle 2.5 for code style checking
- All files must:
  - Start with `#!/usr/bin/env python3`
  - End with a newline
  - Be executable
  - Include proper module, class, and function docstrings
  - Use type annotations
